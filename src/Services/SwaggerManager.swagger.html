DSP Live API

This live API is automatically generated by Swagger using our code and dedicated Swagger configuration files as input. These are all in the main repository and can be augmented to suit your needs. For more developer information and help be sure to check out our documentation and/or wiki.

Important Notes

SSL3 Requirement

In order to keep your data secure we restrict use of our platform and its API to use SSL3 only. If you plan on making requests via cURL, or perhaps your device's Native SDK, you must ensure that the settings are correctly forcing use of SSL3.

For example, using cURL from the command line:

$ curl -3 https://dsp.local/rest/system/user?app_name=curlExample

Authorization: Your API Key

An API key is also required to use our API. This key is your application's name, which can be viewed and changed with the administration application. This key can be sent with each request by one of two methods:

  • Query String: Add the query parameter "app_name=[your key]" to your request replacing [your key] with your actual key.
  • HTTP Header: Add a header called "X-DreamFactory-Application-Name: [your key]" to your request.

VERY IMPORTANT

Future releases of the DreamFactory Services Platform™ (version 2.0.0+) will require the use of the HTTP header X-DreamFactory-Session-Token instead of the methods described above. The value for this header is the session_id retrieved via the authentication steps used. The next section details this process.

We are also researching alternative authorization methods such as OAuth and OpenId.

Authentication: Obtaining a Session ID

If your application is not part of the guest user’s role (or is not hosted on the DSP) then access to any service or data components will require authentication. To authenticate a user, simply make a POST to the /rest/user/session endpoint. The payload you'll send is in JSON and looks like this:

	{“email”:”email_value”, “password”:”password_value”}

Replacing, of course, "email_value" and "password_value" with the credentials of the user for which you wish to authenticate. If successful, the response will contain a session_id value. This indicates that the session has been created successfully.
